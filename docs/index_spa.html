▶ How calibration works
The calibration engine reads outcomes from every settled pick the model has produced (both displayed and shadow-only) and computes a per-sport, per-market realised vs predicted ratio. As sample size grows, each sport × market auto-promotes through phases:
  • Phase 0 — < 100 settled picks: a credibility-weighted share of the target is applied (cred = settled ÷ 100), so a clear miss starts being corrected immediately and ramps up as data grows.
  • Phase A — full single ratio applied to every pick's edge (effective_edge = raw_edge × ratio); applied now equals target.
  • Phase B — per-bucket ratios (50–60% / 60–70% / 70–80% / 80%+) — ≥ 400 settled picks with ≥ 30 per bucket.
  • Phase C — isotonic regression smooth curve (reserved, future build-out).
Target vs Applied: Target is the full measured correction (realised ÷ predicted); Applied is what multiplies edges right now. They're equal from Phase A on; in Phase 0 Applied is a partial share of Target that grows toward it as more picks settle. Both shift daily as new results land.
Reading the ratio: realised / predicted · > 1.0 = model underconfident (real edge bigger than shown), < 1.0 = model overconfident (real edge smaller than shown).

▶Credibility Cap Auto-Adjustment auto-updated · 2026-06-28 16:16:30 UTC
▶ How auto-adjustment works
The system records every pick where the credibility cap fired (capping the model probability toward market). When a cap fires, the shadow log captures both the raw (uncapped) and capped probabilities — so once enough firings have settled, we can compare which prediction was closer to the actual outcome.
  • Value tuning: if raw was more accurate → widen +2.5%; if capped was more accurate → tighten -2.5%; otherwise keep.
  • Mode upgrade: after > 200 cap firings, the system evaluates whether a smooth tanh saturation (Mode 1) would have beaten Mode 0 (hard clip). If tanh wins by > 1.5% MAE, the cap auto-promotes.
Safety bounds: never tighter than ±5.0%, never wider than ±30.0%. Value adjustments throttled to once per 30 days; mode upgrades throttled to once per 60 days. Minimum 50 cap firings required.

▶Closing Line Value
▶ What CLV means
CLV = the market's closing probability minus the probability when the model picked. Positive = the line moved toward our pick after we took it — we beat the close, the strongest known predictor of long-term edge, win or lose. Negative = the market moved against us; a market that does this consistently loses long-term even while winning bets. Between −1% and +1% is noise.
  • Phase 0 — < 30 graded picks: observe only.
  • Phase 1 — 30+: avg CLV ≤ −2% is pulled from the budget pool.
  • Phase 2 — 50+: avg CLV ≤ −1% is pulled.
Gated markets keep being analyzed and graded — only real-money allocation pauses, so an improving market un-gates itself automatically.
